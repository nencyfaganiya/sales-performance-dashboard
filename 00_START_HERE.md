## ❓ Common Questions

### "Do I need to learn everything in the project?"
**No.** You need to:
1. Understand what it does at a high level
2. Be able to walk through one script in detail
3. Explain the business value it demonstrates
4. Answer "why did you make this?" (to show BI skills)

### "What if someone asks about code I don't understand?"
**Be honest.** Say: "Great question. I built this project to demonstrate X skill. Let me walk through the logic..." then explain what you do understand. It's okay to say "I'd need to review that specific function."

### "Should I memorize all the code?"
**No.** Memorize:
- Project purpose and scope
- Key metrics (5,000 records, £24M, etc.)
- Main insights (e.g., "Enterprise customers drive 61% of revenue")
- One example of each: ETL, KPI, visualization

### "How do I talk about this in interviews?"
**Use the 5-minute walkthrough** from `INTERVIEW_PREP_GUIDE.md`:
1. Business context (30 sec)
2. Data pipeline (1 min)
3. KPIs (1 min)
4. Visualizations (1.5 min)
5. Power BI design (1 min)
